## 🚀 570. Managers with at Least 5 Direct Reports

### 💡 Idea

The goal is to find the managers who have **at least 5 direct reports**.

Each employee stores their manager's ID in the `managerId` column. We can first identify managers with **5 or more** direct reports using `GROUP BY` and `HAVING`, then retrieve their names from the `Employee` table.

---

## 💻 Approach 1: `IN` with `GROUP BY`

```sql
SELECT name
FROM Employee
WHERE id IN (
    SELECT managerId
    FROM Employee
    WHERE managerId IS NOT NULL
    GROUP BY managerId
    HAVING COUNT(*) >= 5
);
```

### 🛠️ How It Works

1. Select all non-`NULL` `managerId` values.
2. Group employees by `managerId`.
3. Count the number of direct reports for each manager.
4. Keep only managers with **5 or more** direct reports using `HAVING`.
5. Return the names of those managers using the outer query.

---

## 💻 Approach 2: Self `INNER JOIN`

```sql
SELECT m.name
FROM Employee m
INNER JOIN Employee e
ON m.id = e.managerId
GROUP BY m.id, m.name
HAVING COUNT(*) >= 5;
```

### 🛠️ How It Works

1. Treat the `Employee` table as:
   - `m` → Managers
   - `e` → Employees
2. Join managers with their direct reports using `m.id = e.managerId`.
3. Group the joined records by the manager.
4. Count the number of employees reporting to each manager.
5. Return the managers whose count is **5 or more**.

---

### 📌 Which Approach Should You Use?

- ✅ **Approach 1:** Simple and easy to understand using a subquery.
- ⭐ **Approach 2:** Recommended because it uses an explicit `INNER JOIN`, making the relationship between managers and employees more intuitive. It's also the style most interviewers expect.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| `IN` + `GROUP BY` | `O(n)` | `O(n)` |
| Self `INNER JOIN` | `O(n)` | `O(1)` *(excluding the output result set)* |
