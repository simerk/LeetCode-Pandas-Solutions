## 🚀 608. Tree Node

### 💡 Idea

We need to classify each node in the `Tree` table into one of three types:

* **Root** → The node has no parent (`p_id` is `NULL`).
* **Inner** → The node has at least one child (its `id` appears as a `p_id` of another node).
* **Leaf** → The node is neither a root nor an inner node.

We can use a `CASE` expression to check these conditions in order.

---

### 💻 Solution

```sql
SELECT id,
    CASE
        WHEN p_id IS NULL THEN 'Root'
        WHEN id IN (SELECT DISTINCT p_id FROM Tree) THEN 'Inner'
        ELSE 'Leaf'
    END AS type
FROM Tree;
```

---

### 🛠️ How It Works

1. Iterate through every node in the `Tree` table.
2. If `p_id` is `NULL`, the node has no parent, so it is classified as **Root**.
3. Otherwise, check whether the current `id` appears in the `p_id` column of any other row.
4. If it does, the node has one or more children, so it is classified as **Inner**.
5. If neither condition is true, the node has a parent but no children, making it a **Leaf**.
6. Return the `id` and its corresponding node `type`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(assuming `id` and `p_id` are indexed; otherwise the subquery may increase the cost)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
