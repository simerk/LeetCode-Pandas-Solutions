## 🚀 1158. Market Analysis I

### 💡 Idea

The goal is to report, for every user:

- `user_id`
- `join_date`
- The number of orders they placed in **2019** (`orders_in_2019`)

Since some users may not have placed any orders, we use a **LEFT JOIN** to include all users. We then count only the orders placed during 2019.

---

### 💻 Solution

```sql
SELECT u.user_id AS buyer_id, u.join_date, COUNT(o.order_id) AS orders_in_2019
FROM Users u
LEFT JOIN Orders o
ON u.user_id = o.buyer_id
AND YEAR(o.order_date) = 2019
GROUP BY u.user_id;
```

---

### 🛠️ How It Works

1. Start with the `Users` table so every user appears in the result.
2. Perform a `LEFT JOIN` with the `Orders` table using `buyer_id`.
3. Filter the joined orders to include only those placed in **2019**.
4. Group the results by `user_id` and `join_date`.
5. Count the number of matching orders for each user.
6. Users with no orders in 2019 will have a count of **0**.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single join followed by grouping)*

**Space Complexity:** `O(n)` *(stores grouped user records)*
