## 🚀 1341. Movie Rating

### 💡 Idea

The problem requires finding:

1. The **user** who has rated the greatest number of movies.
   - If multiple users have the same count, return the one with the **lexicographically smallest name**.
2. The **movie** with the highest average rating during **February 2020**.
   - If multiple movies have the same average rating, return the one with the **lexicographically smallest title**.

Since the two results are independent, we solve them separately and combine them using `UNION ALL`.

---

## 💻 Solution

```sql
(SELECT u.name AS results
FROM MovieRating mr
INNER JOIN Users u
ON mr.user_id = u.user_id
GROUP BY u.user_id
ORDER BY COUNT(*) DESC, u.name
LIMIT 1)
UNION ALL
(SELECT m.title AS results
FROM MovieRating mr
INNER JOIN Movies m
ON mr.movie_id = m.movie_id
WHERE mr.created_at BETWEEN '2020-02-01' AND '2020-02-29'
GROUP BY m.movie_id
ORDER BY AVG(mr.rating) DESC, m.title
LIMIT 1)
```

---

### 🛠️ How It Works

#### Part 1: User with the Most Ratings

1. Join `MovieRating` with `Users`.
2. Group the records by user.
3. Count the number of ratings submitted by each user.
4. Sort by:
   - Number of ratings (descending).
   - User name (ascending) to break ties.
5. Return the top user.

#### Part 2: Highest Rated Movie in February 2020

1. Join `MovieRating` with `Movies`.
2. Filter ratings submitted during **February 2020**.
3. Group by movie.
4. Calculate the average rating for each movie.
5. Sort by:
   - Average rating (descending).
   - Movie title (ascending) to break ties.
6. Return the top movie.

#### Final Step

Combine both results into a single column using `UNION ALL`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n log n)` *(grouping followed by sorting in both queries)*

**Space Complexity:** `O(n)` *(stores grouped users and movies)*
