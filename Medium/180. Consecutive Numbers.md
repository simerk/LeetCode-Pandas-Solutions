## 🚀 180. Consecutive Numbers

### 💡 Idea

The goal is to find numbers that appear **at least three times consecutively**.

We can achieve this by performing a **self join** on the `Logs` table. Join each record with the next two consecutive records using their `id` values, then check whether all three records have the same number.

---

## 💻 Approach 1: Self Join (Comma Join Syntax)

```sql
SELECT DISTINCT l1.num AS ConsecutiveNums
FROM Logs l1, Logs l2, Logs l3
where l1.num = l2.num
and l2.num = l3.num
and l1.id = l2.id - 1
and l2.id = l3.id - 1;
```

### 🛠️ How It Works

1. Treat `l1`, `l2`, and `l3` as three consecutive records from the `Logs` table.
2. Perform a self join using the comma join syntax.
3. Match records where `l2` is immediately after `l1`, and `l3` is immediately after `l2`.
4. Check if all three records have the same `num`.
5. Use `DISTINCT` to return each qualifying number only once.

---

## 💻 Approach 2: Self Join (INNER JOIN)

```sql
SELECT DISTINCT l1.num AS ConsecutiveNums
FROM Logs l1
INNER JOIN Logs l2
ON l1.id = l2.id - 1
INNER JOIN Logs l3
ON l2.id = l3.id - 1
WHERE l1.num = l2.num
  AND l2.num = l3.num;
```

### 🛠️ How It Works

1. Perform a self join on the `Logs` table.
2. Treat `l1` as the first record, `l2` as the second, and `l3` as the third consecutive record.
3. Join records whose `id` values are consecutive.
4. Compare the `num` values of all three records.
5. Return the number if it appears three times consecutively.
6. Use `DISTINCT` to avoid duplicate results.

---

### 📌 Which Approach Should You Use?

- ✅ **Approach 1:** Works correctly using the older comma join syntax.
- ⭐ **Approach 2:** Recommended because it uses explicit `INNER JOIN`, making the query more readable, easier to maintain, and aligned with modern SQL best practices.
- 🚀 Both approaches have the **same time and space complexity**, and modern SQL optimizers typically generate similar execution plans.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(with indexing on `id`, the joins are efficient)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
