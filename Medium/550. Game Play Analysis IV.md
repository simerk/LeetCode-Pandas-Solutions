## 🚀 550. Game Play Analysis IV

### 💡 Idea

The goal is to find the **fraction of players who logged in again exactly one day after their first login**.

First, determine each player's **first login date**. Then, check whether the player has another login on the **next day**. Finally, divide the number of such players by the total number of players and round the result to **2 decimal places**.

---

### 💻 Solution

```sql
SELECT
    ROUND(
            COUNT(DISTINCT a.player_id) / (SELECT COUNT(DISTINCT player_id) FROM Activity)
            , 2 ) AS fraction
FROM Activity a
INNER JOIN (
    SELECT player_id, MIN(event_date) AS first_login
    FROM Activity
    GROUP BY player_id
) f
ON a.player_id = f.player_id
AND a.event_date = ADDDATE(f.first_login, 1);
```

---

### 🛠️ How It Works

1. Find each player's first login date using `MIN(event_date)`.
2. Join the result back with the `Activity` table on `player_id`.
3. Check if the player logged in on the day immediately after their first login using `ADDDATE(first_login, 1)`.
4. Count the distinct players who satisfy this condition.
5. Divide this count by the total number of distinct players.
6. Round the result to **2 decimal places**.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single aggregation and join, assuming appropriate indexing)*

**Space Complexity:** `O(n)` *(stores the first login date for each player)*
