## 🚀 1174. Immediate Food Delivery II

### 💡 Idea

The goal is to calculate the percentage of customers whose **first order** was delivered on their **preferred delivery date**.

For each customer:

- Find their **first order**.
- Determine whether it was **immediate** (`order_date = customer_pref_delivery_date`).
- Calculate the percentage of customers whose first order was immediate.
- Round the answer to **2 decimal places**.

There are two common approaches to solve this problem.

---

## 💻 Approach 1: `ROW_NUMBER()` (Recommended)

```sql
SELECT ROUND(AVG(type = 'immediate') * 100, 2) AS immediate_percentage
FROM (
    SELECT customer_id, delivery_id,
        CASE
            WHEN order_date = customer_pref_delivery_date THEN 'immediate'
            ELSE 'scheduled'
        END AS type
    FROM (
        SELECT *,
               ROW_NUMBER() OVER (
                   PARTITION BY customer_id
                   ORDER BY order_date
               ) AS rn
        FROM Delivery ) t
    WHERE rn = 1
) s;
```

### 🛠️ How It Works

1. Rank each customer's orders using `ROW_NUMBER()`.
2. Keep only the first order (`rn = 1`) for each customer.
3. Use a `CASE` statement to classify the order as:
   - `'immediate'`
   - `'scheduled'`
4. The expression `type = 'immediate'` evaluates to:
   - `1` for immediate deliveries.
   - `0` otherwise.
5. Calculate the average and multiply by `100`.
6. Round the result to **2 decimal places**.

---

## 💻 Approach 2: `MIN()` + `INNER JOIN`

```sql
SELECT ROUND( AVG(order_date = customer_pref_delivery_date) * 100, 2) AS immediate_percentage
FROM Delivery d
INNER JOIN (
    SELECT customer_id, MIN(order_date) AS first_order
    FROM Delivery
    GROUP BY customer_id
) t
ON d.customer_id = t.customer_id
AND d.order_date = t.first_order;
```

### 🛠️ How It Works

1. Find the earliest (`MIN`) `order_date` for each customer.
2. Join the result back to the `Delivery` table.
3. Retrieve the customer's first order.
4. Compare `order_date` with `customer_pref_delivery_date`.
5. The comparison returns:
   - `1` for immediate deliveries.
   - `0` for scheduled deliveries.
6. Calculate the average and multiply by `100`.
7. Round the result to **2 decimal places**.

---

### 📌 Which Approach Should You Use?

- ⭐ **Approach 1 (`ROW_NUMBER()`)** is recommended because it clearly identifies the first order for each customer and is easy to extend to similar "first/latest record" problems.
- ✅ **Approach 2 (`MIN()` + `INNER JOIN`)** is also an excellent solution and is preferred when window functions are unavailable.

---

### ⏱️ Complexity

| Approach | Time | Space |
|----------|------|-------|
| `ROW_NUMBER()` | `O(n log n)` | `O(n)` |
| `MIN()` + `INNER JOIN` | `O(n)` | `O(n)` |
