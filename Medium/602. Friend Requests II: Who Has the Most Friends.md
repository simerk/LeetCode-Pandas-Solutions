## 🚀 602. Friend Requests II: Who Has the Most Friends

### 💡 Idea

The goal is to find the person who has the **highest number of friends**.

Each friendship appears as a request between two users (`requester_id` and `accepter_id`). Since friendship is **bidirectional**, we need to count each user's appearances in **both columns**. We combine them using `UNION ALL`, count the occurrences of each user, and return the user with the highest count.

---

### 💻 Solution

```sql
SELECT id, COUNT(*) AS num
FROM (
    SELECT requester_id AS id FROM RequestAccepted
   UNION ALL
   SELECT accepter_id AS id FROM RequestAccepted
) AS t
GROUP BY id
ORDER BY num DESC
LIMIT 1;
```

---

### 🛠️ How It Works

1. Select all `requester_id` values.
2. Select all `accepter_id` values.
3. Combine both result sets using `UNION ALL` so every friendship is counted.
4. Group the combined data by `id`.
5. Count the number of occurrences for each user.
6. Sort the counts in descending order.
7. Return the user with the highest number of friends using `LIMIT 1`.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(linear scan of the table followed by grouping)*

**Space Complexity:** `O(n)` *(stores the combined IDs before grouping)*
