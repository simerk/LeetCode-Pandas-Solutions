## 🚀 197. Rising Temperature

### 💡 Idea

The goal is to find the dates where the temperature is **higher than the previous day's temperature**.

Since the weather data is stored in the same table, we can use a **self join** to compare each day's temperature with the temperature of the previous day.

---

### 💻 Approach 1: Self Join (Comma Join Syntax)

```sql
SELECT
    a.id
FROM Weather a,
     Weather b
WHERE a.temperature > b.temperature
AND a.recordDate = ADDDATE(b.recordDate, 1);
```

### 🛠️ How It Works

1. Treat `a` as the current day's record and `b` as the previous day's record.
2. Perform a self join using the comma join syntax.
3. Match records where `a.recordDate` is exactly one day after `b.recordDate`.
4. Compare the temperatures of the two days.
5. Return the `id` if the current day's temperature is higher.

---

### 💻 Approach 2: Self Join (INNER JOIN)

```sql
SELECT
    w1.id
FROM Weather w1
INNER JOIN Weather w2
ON w1.recordDate = ADDDATE(w2.recordDate, 1)
WHERE w1.temperature > w2.temperature;
```

### 🛠️ How It Works

1. Perform a self join using `INNER JOIN`.
2. Treat `w1` as the current day's record and `w2` as the previous day's record.
3. Match records where `w1.recordDate` is one day after `w2.recordDate`.
4. Compare the temperatures of the matched records.
5. Return the `id` if the current day's temperature is higher.

---

### 📌 Which Approach Should You Use?

- ✅ **Approach 1:** Works correctly but uses the older comma join syntax.
- ⭐ **Approach 2:** Recommended because it uses explicit `INNER JOIN`, making the query more readable and aligned with modern SQL best practices.

---

### ⏱️ Complexity

**Time Complexity:** `O(n²)` *(depends on the self join and indexing on `recordDate`)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
