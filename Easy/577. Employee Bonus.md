## 🚀 577. Employee Bonus

### 💡 Idea

We need to find employees whose bonus is **less than 1000** or who **did not receive a bonus**.

A `LEFT JOIN` is used to include all employees. If an employee doesn't have a matching record in the `Bonus` table, the bonus fields will be `NULL`, allowing us to include them in the result.

---

### 💻 Solution

```sql
SELECT e.name, b.bonus
FROM Employee e
LEFT JOIN Bonus b
ON e.empId = b.empId
WHERE b.bonus < 1000 OR b.empId IS NULL;
```

---

### 🛠️ How It Works

1. Start with the **Employee** table.
2. Perform a `LEFT JOIN` with the **Bonus** table using `empId`.
3. If an employee has a bonus, retrieve its value.
4. If no matching bonus record exists, the bonus columns are returned as `NULL`.
5. Filter employees whose bonus is **less than 1000** or whose bonus record is **missing**.
6. Return the employee's name and bonus.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(depends on the join operation and indexing on `empId`)*

**Space Complexity:** `O(1)` *(excluding the output result set)*
