## 🚀 1280. Students and Examinations

### 💡 Idea

The goal is to report the number of times each student attended each exam.

Since every student should appear for **every subject**, we first generate all possible **student-subject combinations** using a `CROSS JOIN`. Then, we `LEFT JOIN` the `Examinations` table to count how many times each student attended each subject's exam.

---

## 💻 Solution

```sql
SELECT
    s.student_id,
    s.student_name,
    sub.subject_name,
    COUNT(e.subject_name) AS attended_exams
FROM Students s
CROSS JOIN Subjects sub
LEFT JOIN Examinations e
ON s.student_id = e.student_id
AND sub.subject_name = e.subject_name
GROUP BY s.student_id, s.student_name, sub.subject_name
ORDER BY s.student_id, sub.subject_name;
```

---

### 🛠️ How It Works

1. Use a `CROSS JOIN` to generate every possible student-subject combination.
2. `LEFT JOIN` the `Examinations` table using both `student_id` and `subject_name`.
3. Match the examination records for each student and subject.
4. Group the results by student and subject.
5. Use `COUNT()` to count the number of matching examination records.
6. Students who never attended an exam for a subject will have a count of **0**.

---

### ⏱️ Complexity

**Time Complexity:** `O(s × sub + e)`  
*(where `s` = number of students, `sub` = number of subjects, and `e` = number of examination records)*

**Space Complexity:** `O(s × sub)` *(stores all student-subject combinations)*
