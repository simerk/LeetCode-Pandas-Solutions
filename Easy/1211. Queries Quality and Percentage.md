## 🚀 1211. Queries Quality and Percentage

### 💡 Idea

The goal is to calculate, for each `query_name`:

- **Quality** = average of `rating / position`
- **Poor Query Percentage** = percentage of queries with a rating less than `3`

We group the records by `query_name` and use aggregate functions to compute both metrics. MySQL treats boolean expressions as `1` (TRUE) and `0` (FALSE), allowing us to calculate the poor query percentage using `AVG()`.

---

## 💻 Solution

```sql
SELECT
    query_name,
    ROUND(AVG(rating / position), 2) AS quality,
    ROUND(AVG(rating < 3) * 100, 2) AS poor_query_percentage
FROM Queries
GROUP BY query_name;
```

---

### 🛠️ How It Works

1. Group the records by `query_name`.
2. Calculate the average of `rating / position` using `AVG()` to obtain the query quality.
3. Evaluate the expression `rating < 3`:
   - `TRUE` becomes `1`.
   - `FALSE` becomes `0`.
4. Calculate the average of these boolean values to obtain the fraction of poor queries.
5. Multiply by `100` to convert it into a percentage.
6. Round both values to **2 decimal places**.

---

### ⏱️ Complexity

**Time Complexity:** `O(n)` *(single scan with grouping)*

**Space Complexity:** `O(n)` *(stores grouped query names)*
